# Enterprise UX Patterns（参考金蝶/用友风格）

## A. 布局与信息密度

- 页面采用 8px 基础栅格，保证模块间距一致。
- 顶部保留“面包屑 + 页面标题 + 主操作区”。
- 高密度页面优先使用卡片分组或折叠面板，避免长滚动无分层。
- 表格页默认展示“关键业务字段 + 状态 + 更新时间 + 操作”。

## B. 列表与表格

- 提供“快速筛选 + 高级筛选”双层机制。
- 操作列遵循“高频操作前置，低频操作收纳到更多菜单”。
- 批量操作必须显示影响条数，并提供撤销或二次确认。
- 固定列建议：首列关键标识，末列操作。

## C. 表单与录入

- 按业务语义分组字段：基础信息、策略参数、通知配置。
- 必填项在标签处显式标识，不仅在错误时提示。
- 错误提示优先给“可执行建议”，避免只说“参数不合法”。
- 长表单支持“分段保存”或“草稿保存”。

## D. 状态与反馈

- 使用稳定语义色：成功（绿）、警告（黄）、危险（红）、进行中（蓝/灰）。
- 异步任务必须有可追踪状态（排队中/执行中/完成/失败）。
- 对不可逆操作给出影响范围与审计提示。
- 全局通知（toast）只用于轻反馈；关键结果应在页面主体可见。

## E. 文案规范

- 按钮文案用动词开头：保存、提交、发布、撤回。
- 确认框标题写“动作 + 对象”，正文写“影响 + 后果 + 是否可恢复”。
- 错误文案包含：错误原因、建议操作、支持联系路径（如有）。

## F. 可用性红线

- 不隐藏关键权限状态（禁用原因必须可见）。
- 不让用户在未知后果下执行危险动作。
- 不在一个页面混用多套状态语义或颜色语义。
- 不让同一类任务在不同页面出现相反的交互模式。
